Microsoft and Java?

  • It's a new world
  • Microsoft has embraced open source

VS Code

  • Based on Electron
  • Written in TypeScript/Node.js
  • Project Headed by Erich Gamma (GoF, Eclipse, etc.)
  • Open Source on GitHub
  • Configured and extended through Extensions (plugins)

Basic Setup

This is my setup for today...

  • Install VS Code from https://code.visualstudio.com/
  • Install Git from https://git-scm.com/
  • Install a JDK from http://java.oracle.com/
  • Download and unzip Maven from http://maven.apache.org/
  • Download and unzip Tomcat from http://tomcat.apache.org/
  • Set JAVA_HOME, and add JAVA_HOME/bin to the path

How does it work?

  • It's Eclipse under the covers
  • RedHat implemented a Language Server for Java, using Eclipse
  • github.com/Microsoft/language-server-protocol
  • Language server is an interface for
    • Intellisense
    • Basic Refactorings
    • etc.
  • Many of the extensions simply call out to the shell

What We'll See

  • Intellisense, Code Lens in Java Code
  • JUnit Support
  • Debug Java Code
  • Spring Boot Support
  • Tomcat Support

What's Missing?

  • Full refactoring
  • Wizards - new class, new interface, etc.
  • Good integration between the different extension (all 0.x versions currently)
  • Feels a little like the early days of hacking Eclipse - takes a lot of setup
